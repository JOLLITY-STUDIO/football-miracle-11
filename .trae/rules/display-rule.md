# 球员卡片显示规则

## 图标类型说明

| 图标名称 | 说明 |
|---------|------|
| 射门（足球） | 进攻/足球图标 |
| 防守 | 防守/盾牌图标 |
| 逼抢（压迫） | 压迫/逼抢图标 |
| 协同（+号） | 配合/协同图标（半圆内有+号） |

## 位置说明

每张球员卡有6个半圆图标位置：
- 顶部25%
- 顶部75%
- 左侧50%
- 右侧50%
- 底部25%
- 底部75%

## 技能说明

| 技能名称 | 说明 |
|---------|------|
| 即时效果 | 立即生效的技能效果 |
| 去掉对手手牌一张 | 对手随机弃掉一张手牌 |
| 抽2选一 | 抽两张牌选一张保留，另一张弃掉 |
| 抽一张卡 | 抽一张牌 |
| 突破防守 | 无视防守进行攻击 |
| 射门 | 进行射门 |
| 压迫 | 施加压迫效果 |
| 无视所有防守 | 射门时无视所有防守 |

为每个阶段实现对决过场系统 duel system 来展示结算过程，实体桌游的游戏规则我们都有的，每个阶段每个步骤我们在电子系统的实现肯定会考虑每个阶段的计算和步骤的动作过场的，比如每个阶段开始时展示当前阶段的信息，每个步骤开始时展示当前步骤的信息，每个步骤结束时展示结算结果，每个阶段结束时展示阶段总结。并且是可以查看操作过程和结算结果的，如果没有对决只是一方的操作信息，也是都记录的，保证这个规则是完整合理的，而不是只看结果不看过程。
建立统一的规范，比如UI，代码结构等。然后在合格文件备注相关信息，并进行查看。不断优化和修正，直到确定最终完整覆盖了整个游戏开发范围就停止更新。结束方法自己定下来。看到标识自动结束更新。
