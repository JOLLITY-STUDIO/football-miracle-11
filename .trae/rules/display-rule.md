牌套问题
游戏内卡牌尺寸为66x43mm，所有卡牌大小是一样的，卡牌放置的位置需要考虑卡牌大小保持一致性，还有球场的大小跟卡牌的长宽有关，一张卡横放是会占用球场2格位置的。协同区放置卡牌的位置也要注意这一点
协同区放置协同卡的位置是共享进攻或防守的，都是在同一个位置盖放。直到对局阶段，才有可能翻转卡牌显示星数累加力量
gen\截图\卡牌\协同卡\协同卡盖放示意图.png 你看下协同卡是这么盖放的，进攻和防守都是共享这个卡区的，不需要分放置在进攻还是防守，只要知道那边是进行射门阶段就知道谁是进攻，谁是防守，一个玩家就有三个放置区，根据控制权位置不同，分别对应不同的可放置区。
所有卡应该有个基类，就是白板卡组件，基础就是固定的长宽，所有放置区域卡牌区域的都跟他的大小绑定

rules.MD是这么描述的：
### 4. 版图与区域设置
1. 将游戏版图放置在桌面中间。
2. 坐在“主场”一侧的玩家为主场球队，另一侧为客场球队。（这里我们实现为投币确定是主客场球队，而不是根据位置确定，简化的话我们可以直接更改home 或者 away 的的位置和属性来体现出来也可以））
3. 将控制标记放置在控制轨道的中间位置。（电子版就是重置居中，这些都要有动画动作来实现这个重置效果，而不是突然就出现在那里，最好有3d下棋的感觉那种落）
4. 分别设置协同卡牌堆和点球卡牌堆。
5. 将所有其他配件放置在游戏主板图旁，方便拿取。
这里描述的都是实体桌游的动作，那我们好游戏里面也要提现这个动作而不是直接展示出来，你能明白我的意思吗，我的意思是我的游戏是要“动起来”的感觉的，这才叫游戏，这才叫身临其境，每个动作都有对应的音效，比如球队行动，球员行动，玩家射门，玩家协同，玩家得分（欢呼庆祝，个人声优）等，这些都要在游戏中提现出来，而不是直接展示出来。
关于球员：主场和客场的基础卡牌是433阵型，可以在这个目录查阅，gen\截图\卡牌\球员，目前我看到官方搭设定时，后腰和左右后卫都有即时效果，后腰有抽2选1技能，边后卫都是抽一张协同卡的技能,其他中场没有技能，中后卫也没有技能，中锋也没有技能，左右边锋有突破防守技能
控制权的展示效果，红色表示攻势1格，绿色表示普通正常2格，蓝色表示守势2格，控制权的变化是双向的，一边进1另一边就退1，如此类推，完全看的清楚双方当前控制权的状态，方便对应协同卡，和思考接下来采取的措施
