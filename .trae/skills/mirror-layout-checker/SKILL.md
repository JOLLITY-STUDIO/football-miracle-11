# 布局镜像检查与修复

本技能用于自动化检查并修复界面中“玩家/对手”与“上下分区”的镜像布局一致性问题，确保三块区域（左侧面板、中心球场、右侧面板）在尺寸比例、边框方向、渐变方向、3D 厚度与视觉元素上保持对称与镜像。

## 何时调用
- 需要验证是否满足镜像布局规范时
- 发现左右或上下区域视觉不一致，要求自动生成修复补丁时
- 进行组件化或样式重构后进行一致性回归检查时

## 检查范围
- 尺寸比例是否为 25.2% / 49.6% / 25.2%
- 左右面板的边框与渐变方向是否相对（左侧 border-r/渐变向右，右侧 border-l/渐变向左）
- 中央球场的线条与中轴是否对称，附加材质不影响交互
- 3D 厚度与角件（如螺钉）是否成镜像分布
- 交互元素（牌堆/弃牌/抽卡区）是否保持上下分区一致的语义位置

## 使用步骤
1. 搜索并定位三块组件文件：
   - src/components/LeftPanel.tsx
   - src/components/CenterField.tsx
   - src/components/RightPanel.tsx
2. 校验以下关键样式与结构：
   - 左右面板：basis、border-*、bg-gradient-* 方向、transform-style-3d
   - 中心球场：basis、场地线条/中圈/中线的对称性
   - 额外装饰（阴影、材质、角件）的左右/上下镜像
3. 如发现不一致，生成最小化修复补丁（保持功能不变，优先修改样式）
4. 运行类型检查与本地预览，确保无错误并在浏览器验证

## 产出
- 一组修复建议与补丁（apply_patch 友好）
- 验证结果：类型检查通过、预览运行、镜像检查通过

## 示例调用
- “检查当前实现是否满足镜像规则并自动修复”
- “左/右面板的渐变方向不一致，请修复为镜像”
- “抽卡区与右侧面板位置不统一，调整至一致且镜像”

## 注意
- 不修改业务逻辑与交互事件，仅限视觉层面的镜像修复
- 保持现有组件化与命名规范，补丁尽量小且可读
