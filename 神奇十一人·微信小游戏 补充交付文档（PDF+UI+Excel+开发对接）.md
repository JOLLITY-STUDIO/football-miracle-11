### 二、核心玩法与规则

#### 2.1 核心机制

- 布阵：球员卡按前锋/中场/后卫放置在对应区域

- 对抗：进攻力量 vs 防守力量，数值高者获胜

- 协同：使用协同卡强化攻防效果

- 状态：控制轨道决定协同卡使用数量

- 事件：铲球、点球、换人、伤停补时

#### 2.2 阵容配置

- 首发阵容：前锋3张 + 中场4张 + 后卫3张

- 替补阵容：3张

- 换人限制：单局最多3次换人

#### 2.3 场地区域（4线制）

1. 前线：仅前锋可放置

2. 二线：前锋、中场可放置

3. 三线：中场、后卫可放置

4. 四线：仅后卫可放置

#### 2.4 回合流程

1. 球队行动：抽取协同卡 / 移动控制标记

2. 球员行动：放置球员卡 / 发起进攻射门

3. 攻防结算：系统自动计算数值并判定结果

4. 回合结束，切换至对方行动

#### 2.5 攻防结算规则

- 进攻力量 = 球员进攻图标 + 协同卡加成

- 防守力量 = 对方球员防守图标

- 进攻力量 ＞ 防守力量 → 进球得分

#### 2.6 协同卡使用规则

- 进攻态：最多使用3张协同卡

- 常态：最多使用2张协同卡

- 防守态：不可使用协同卡

#### 2.7 特殊事件规则

- 铲球卡：触发点球大战

- 点球：双方各选择1张点球卡，点数大的一方进球

- 伤停补时：协同卡牌库耗尽后开启，仅可执行进攻操作

- 中场休息：可进行换人调整，下半场交换先手

### 八、风险控制与避坑方案

1. 规则过于复杂：全部计算、判定由系统完成，玩家仅做选择

2. 操作繁琐：全程仅保留点击操作，无复杂手势

3. 单局时长过长：流程简化，控制在5–10分钟

4. 新手理解成本高：高亮提示+气泡引导+极简文字说明

5. 开发周期过长：优先实现核心玩法，迭代功能后置
